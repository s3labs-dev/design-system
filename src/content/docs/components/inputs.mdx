## Input Anatomy

| Element | Style |
|---------|-------|
| Background | `--s3-void` |
| Border | 1px `--s3-moss-dark` |
| Border Radius | 12px |
| Text Color | `--s3-cream` |
| Placeholder | `--s3-moss` |
| Focus Border | `--s3-bio-green` |
| Focus Glow | 3px rgba(126, 217, 87, 0.15) |

## CSS Implementation

```css
.input-wrapper {
  display: flex;
  flex-direction: column;
  gap: var(--s3-space-2);
}

.input-label {
  font-size: 13px;
  font-weight: 500;
  color: var(--s3-cream-dim);
}

.input {
  padding: var(--s3-space-3) var(--s3-space-4);
  font-family: var(--s3-font-body);
  font-size: 14px;
  background: var(--s3-void);
  border: 1px solid var(--s3-moss-dark);
  border-radius: var(--s3-radius-md);
  color: var(--s3-cream);
  transition: all var(--s3-duration-fast) var(--s3-ease-out);
}

.input:focus {
  outline: none;
  border-color: var(--s3-bio-green);
  box-shadow: 0 0 0 3px rgba(126, 217, 87, 0.15);
}

.input::placeholder {
  color: var(--s3-moss);
}

/* Disabled */
.input:disabled {
  opacity: 0.5;
  cursor: not-allowed;
}

/* Error */
.input-error {
  border-color: var(--s3-error);
}

.input-error:focus {
  box-shadow: 0 0 0 3px rgba(239, 107, 107, 0.15);
}
```

## Accessibility

- Labels are associated with inputs via `for`/`id` attributes
- Focus states are clearly visible with the Bio-Green glow
- Error states use both color and text to indicate issues
- Disabled inputs maintain semantic meaning while appearing inactive
